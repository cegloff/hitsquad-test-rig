<project_specification>
Create a **complete, fully playable Klondike Solitaire** game as a **single self-contained HTML file** (everything — HTML, CSS via Tailwind CDN, and pure vanilla JavaScript — must be inside one file; no external images, no other CDNs except Tailwind, and it must work offline after the initial load).

### Game Rules (Standard Klondike – Draw 3)
- 52-card French deck (no jokers).
- 7 tableau columns: leftmost starts with 1 card, increasing to 7 on the right. Only the top card of each pile is face-up initially.
- 4 foundation piles (Ace → King, one per suit).
- Stock pile (draws 3 cards at a time to the waste pile). Only the top card of the waste is playable.
- When stock is empty, clicking the empty stock area recycles the entire waste pile back to stock (face-down, unlimited passes).
- Valid moves:
  - Tableau: descending rank, alternating colors (red/black). You can move an entire correctly sequenced build at once.
  - Empty tableau column can only accept a King (or a build starting with a King).
  - Foundations: ascending rank, same suit.
- Double-clicking any card automatically moves it to a valid foundation if possible.
- Include a toggle in the UI to switch between "Draw 3" and "Draw 1" modes.

### Visual Style (Must be stunning)
- Dark cyberpunk/neon theme: deep background (#0a0a1f or similar) with subtle radial gradient "felt" effect.
- Bold, vibrant colors:
  - Hearts & Diamonds: electric red (#ff2d55)
  - Spades & Clubs: bright cyan/white (#64f0ff / #e0f2fe)
- Heavy **bloom / neon glow effects** everywhere:
  - Moving or selected cards get multi-layer box-shadow glow (cyan + magenta + white).
  - Valid drop targets glow brightly.
  - Card backs are premium dark with subtle neon pattern and inner glow.
- Cards: rounded corners (8px), large Unicode suits (♥ ♦ ♣ ♠), big rank in corners and center, smooth flip animation.
- Smooth CSS transitions and animations for dealing, moving, and flipping cards.

### Layout & Responsiveness (Critical)
- Fully responsive – looks perfect on desktop (classic layout) and mobile (portrait & landscape).
- Desktop layout:
  - Top bar: title, timer, moves counter, New Game button, Draw-mode toggle.
  - Top game row: Stock + Waste (left) | 4 Foundations (right).
  - Below: 7 Tableau columns with proper vertical stacking/offset.
- Mobile: cards automatically scale (use clamp() + Tailwind responsive classes), generous touch targets, layout stays clear.
- Use Tailwind CSS via CDN (`<script src="https://cdn.tailwindcss.com"></script>`) for fast, clean styling.

### Interactions & Features
- Full mouse drag-and-drop (with ghosting of the entire build being moved).
- Full touch support for mobile (drag with finger).
- Click-to-select then click-to-place as backup.
- Visual feedback: highlight valid drop targets while dragging.
- Undo button (store at least the last 8 moves).
- Timer and move counter.
- New Game button (with nice shuffle animation).
- Win detection: when all foundations are complete, show a beautiful full-screen win modal with confetti (simple canvas particle explosion) + stats + "Play Again".
- Hint button (highlights one legal move).

### Technical Requirements
- Pure vanilla JavaScript (no frameworks).
- Cards represented as clean objects: `{ id, suit, rank, faceUp }`.
- Proper game state management (tableau as arrays, foundations, stock, waste).
- Z-index and absolute positioning for realistic card stacking.
- Well-commented, readable code with clear sections (constants, game logic, rendering, event handlers).
- No console errors, no lag even on mobile.

Make it look and feel premium, addictive, and visually spectacular with the requested dark neon bloom aesthetic
